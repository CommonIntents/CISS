## 一、核心定位

CISS（CIS Secure）是**基于mTLS的传输安全实现**。

它在Agent与工具之间建立端到端加密信道，并在连接建立的第一毫秒完成密码学身份证明。

CISS是协议栈的**骨骼**——提供结构性的安全保障。

---

## 二、与其他层的关系

CIB是**规范**，CISS是**实现**。

CIB定义：“意图数据通过协商后的格式，承载于安全的传输信道之上。”CISS实现：“这条安全信道是mTLS over HTTPS。”

CISS提供**长期身份证明**。CAP在此之上签发**短生命周期操作凭据**（JWT）。长期身份不直接用于操作授权，操作授权必须通过有时效的凭据。

---

## 三、身份模型

### 3.1 信任锚

Agent的私钥是其身份的唯一信任锚。不依赖任何中心化机构签发的令牌。

Agent的身份由其持有的私钥所对应的证书公钥确定。证书可以是CA签发的标准X.509证书，也可以是自签名证书。无论证书来源如何，Agent的身份在mTLS握手完成时即被密码学证明。

### 3.2 身份验证流程

```
Agent                         服务端
  │                              │
  ├──── 建立TCP连接 ────────────►│
  ├──── TLS ClientHello ────────►│
  │                              │
  │◄─── TLS ServerHello ────────┤
  │      + 服务端证书             │
  │◄─── CertificateRequest ─────┤
  │      (要求客户端证书)         │
  │                              │
  ├──── 客户端证书 ──────────────►│
  ├──── CertificateVerify ──────►│
  │      (用私钥签名握手数据)     │
  │                              │
  │◄─── 握手完成 ───────────────┤
  │      Agent身份已密码学证明    │
  │                              │
  ├──── 加密的CIS/CAP数据 ───────►│
```

连接建立的第一毫秒即完成身份绑定。任何后续操作都绑定到此已证明身份。

## 四、mTLS握手规范

### 4.1 证书要求

服务端和客户端均须持有证书。证书公钥作为身份标识。

### 4.2 证书验证

服务端验证客户端证书的有效性：证书链完整、证书在有效期内、证书未被吊销。

Agent验证服务端证书的有效性：防止中间人攻击。

### 4.3 会话管理

TLS会话可缓存以加速后续连接。会话缓存的管理和有效期由实现决定。

---

## 五、协议边界

CISS **负责**：
- 定义mTLS握手规范
- 定义基于私钥的身份模型
- 定义证书验证要求

CISS **不负责**：
- 规定TLS的具体版本（由运维配置决定）
- 规定是否启用0-RTT（由运维配置决定）
- 规定证书的签发和吊销流程（由PKI生态提供）
- 定义传输格式协商（由CIB负责）
- 定义基于身份的授权逻辑（由CAP负责）

## 六、安全属性

| 属性 | 实现方式 |
|:---|:---|
| **加密** | TLS端到端加密，数据在传输中不可窃听 |
| **身份** | 客户端证书验证，Agent身份在握手时确立 |
| **完整性** | TLS记录层MAC，篡改即丢弃连接 |
| **前向安全性** | 由TLS密钥交换算法保证（取决于运维配置） |

---

## 七、证书管理

### 7.1 证书签发

证书由PKI生态提供。可使用企业内部CA、公共CA或自签名证书。协议不规定签发方式。

### 7.2 证书吊销

证书吊销通过PKI生态的标准机制处理（CRL或OCSP）。协议不规定吊销方式。

### 7.3 密钥保护

Agent私钥的存储和保护由实现负责。建议使用安全硬件模块或操作系统密钥链。协议不规定保护方式。

---

## 八、与CIB的关系

CIB将CIS绑定到CISS。当CIB协商传输格式和完整性校验后，实际的数据传输通过CISS的mTLS信道完成。

CISS是CIB当前绑定的传输实现。未来可能出现CISS-QUIC、CISS-PQC等替代实现。所有替代实现必须提供相同的安全属性：端到端加密、握手时身份证明、传输完整性保护。

---

## 九、协议边界再确认

CISS是协议栈中负责**传输安全**的一层。它不定义意图语义（CIS的责任），不定义授权逻辑（CAP的责任），不定义格式协商（CIB的责任）。

**CISS的存在，是为了让CAP和CIS不需要关心信道是否安全——CISS保证了它。**
